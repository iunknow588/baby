# 实施计划 v0.0.1

## Phase 1：最小可用控制层（1-2 天）
1. 建立 `workflow-registry` 配置文件（先支持 2-3 条主流程）。
2. 在响应中强制补齐 `route/workflowId/workflowVersion/traceId`。
3. 前端统一展示 `flow_first` 规划消息 + 最终执行消息。

验收标准：
1. 任一请求都能看到明确路由与流程步骤。
2. 历史消息不再落库工具中间态占位文案。

## Phase 2：执行网关与稳定性（2-4 天）
1. 抽象 `execution-gateway`，统一 Coze 与本地执行器接口。
2. 增加超时/重试/熔断策略与降级原因编码。
3. 增加配置启动校验：缺关键变量直接启动失败并报警。

验收标准：
1. 缺少 `COZE_*` 时启动即失败，不进入可服务状态。
2. 每次降级都有可追踪错误码与 traceId。

## Phase 3：观测与回归（2-3 天）
1. 建立 trace 日志聚合与查询视图。
2. 建立路由回归集（覆盖书法、作文、心理辅导、通用问答）。
3. 联动自动化测试：协议测试 + 集成 smoke。

验收标准：
1. 新版本发布前可自动发现“路由漂移/工作流错绑”。
2. 关键链路具备可回放能力。

## Phase 4：CozeLoop 深化对接（可选）
1. 将评测集与实验流程接入发布门禁。
2. 将 Prompt 版本与 workflow 版本建立映射关系。

验收标准：
1. 发布记录包含：Prompt 版本、workflowVersion、回归结果摘要。
